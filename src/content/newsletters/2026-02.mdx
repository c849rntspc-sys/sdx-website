## API Now Available for Third-Party Integrators

The SDX RESTful API is now open for third-party software providers. Integration partners can authenticate via scoped API keys with 11 granular permission scopes — from `properties:read` to `reports:write`. Keys are shown once at creation, stored as bcrypt hashes, rate-limited, and instantly revocable. Full documentation is available in the [Support Center](/docs).

## Multi-Actor Access Model

SDX now supports three distinct user types: Direct Owners who manage their own portfolios, Authorized Representatives (consultants and advisors) who manage portfolios on behalf of multiple owners, and API Integration Partners who push and pull data programmatically. Each actor type has tailored capabilities, granular permissions, and a complete audit trail.

## Data Dividend Methodology Published

The Commercialization Committee has published the data dividend calculation methodology. The formula weights four variables: number of buildings, total floor area, data quality score, and data completeness. This model fairly treats owners of all sizes and asset classes — from a single-building owner to the largest institutional portfolios. One dividend per building per underlying owner, regardless of how many software platforms submit that data.

## Governance Update

The Product Committee and Commercialization Committee have held their first quarterly sessions. Two Board members co-chair each committee. Committee members serve 2-year terms, renewable by the Board.

## What's Next

March will bring expanded compliance framework support including GRESB 2026 alignment, NYC LL97 reporting templates, and Boston BERDO integration. We are also onboarding the first wave of licensed content buyers.
